## üìã Required Environment Variables for Firebase:

Your `.env.local` file should have:

```bash

FIREBASE_ADMIN_PROJECT_ID=justforview1
FIREBASE_ADMIN_CLIENT_EMAIL=firebase-adminsdk-fbsvc@justforview1.iam.gserviceaccount.com
FIREBASE_ADMIN_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...your-key...\n-----END PRIVATE KEY-----\n"
```

## üîç Verification Steps:

1. **Restart your development server** (the environment variables are only loaded at startup):

   ```bash
   # Stop the current server (Ctrl+C)
   npm run dev
   ```

2. **Check if the variables are being read**:

   - The error should now be more specific if any variable is missing
   - You should see clear error messages like "FIREBASE_ADMIN_PROJECT_ID is not set"

3. **Common Issues**:
   - ‚ùå **Private key format**: Make sure the private key is wrapped in quotes and has `\n` for line breaks
   - ‚ùå **Missing SESSION_SECRET**: Also add this to your `.env.local`:
     ```bash
     SESSION_SECRET=your_secure_random_string_at_least_32_characters_long
     ```
   - ‚ùå **Server not restarted**: Environment variables are only loaded when the server starts
